# 💾 Инструкция по работе с базой данных

Описание структуры ключевых таблиц PostgreSQL, используемых в приложении **DesktopBrowser**.

---

## 1. 🔌 Таблица `connects`

Хранит данные для авторизации:

| Поле       | Назначение                                                  | Пример                          |
|------------|-------------------------------------------------------------|---------------------------------|
| phone      | Номер телефона РФ без +7, используется для входа в ЛК       | `9998887766`                    |
| proxy      | Прокси: единый IP для всех действий в ЛК                    | `http://user:pass@1.2.3.4:5678` |
| mail       | Yandex-почта для авторизации в Ozon и Yandex.Маркет         | `example@yandex.ru`             |
| token      | Токен IMAP/SMTP от Yandex для получения писем (Ozon)        | `2a9***81f`                     |
| pass_mail  | Пароль от Yandex-почты (исп. для Yandex.Маркет)             | `mypassword123`                 |

📘 [Инструкция по настройке Yandex-почты](setup_yandex_mail.md)

---

## 2. 🏬 Таблица `markets`

Хранит информацию о компании и её связке с маркетплейсом:

| Поле         | Назначение                                               | Пример        |
|--------------|----------------------------------------------------------|---------------|
| marketplace  | Название платформы (из `marketplaces`)                   | `Ozon`        |
| name_company | Название компании для отображения в интерфейсе           | `MyCompany`   |
| phone        | Телефон из таблицы `connects`, для связи с магазином     | `9998887766`  |
| entrepreneur | ИП / юр.лицо, используется во внутренней части           | `ИП Иванов`   |
| client_id    | Идентификатор клиента, используется во внутренней части  | `34243`       |

---

## 3. 👥 Таблицы `group_table` и `group_market`

### ➕ Группа (таблица `group_table`)

Хранит доступные группы пользователей:

| Поле   | Назначение                    | Пример                   |
|--------|-------------------------------|--------------------------|
| group  | Название группы               | `Manager WB`             |
| comment| Описание, комментарий         | `Менеджеры Wildberries`  |

### 🔗 Привязка к магазинам (таблица `group_market`)

Определяет, к каким компаниям и маркетплейсам привязана группа:

| Поле         | Назначение                                          | Пример            |
|--------------|-----------------------------------------------------|-------------------|
| group        | Название из `group_table`                           | `Manager WB`      |
| marketplace  | Название платформы                                  | `WB`              |
| name_company | Название компании из `markets`                      | `MyCompany`       |

---

## 4. 👤 Таблица `users`

Добавляет нового пользователя для авторизации в приложении:

| Поле     | Назначение                                 | Пример         |
|----------|--------------------------------------------|----------------|
| user     | Логин                                      | `manager1`     |
| password | Пароль                                     | `secure_pass`  |
| name     | Имя пользователя (отображается в UI)       | `Иван`         |
| group    | Название группы (из `group_table`)         | `Manager WB`   |

---

## 5. 📋 Таблица `log`

Используется для хранения логов всех действий и событий:

| Поле        | Назначение                                            | Пример                 |
|-------------|-------------------------------------------------------|------------------------|
| timestamp   | Дата и время события                                  | `2024-04-01 14:23:11`  |
| action      | Тип действия: `INFO`, `ERROR`, `WARNING` и т.д.       | `INFO`                 |
| user        | Пользователь                                          | `manager1`             |
| ip_address  | IP-адрес клиента                                      | `94.24.13.18`          |
| city        | Город, полученный по IP                               | `Moscow`               |
| country     | Страна                                                | `Russia`               |
| proxy       | Используемый прокси                                   | `http://ip:port`       |
| description | Текст описания события                                | `Успешный вход`        |

---

## 6. 📲 Таблица `phone_message`

Хранит информацию о процессе авторизации по коду:

| Поле          | Назначение                                               | Пример               |
|---------------|----------------------------------------------------------|----------------------|
| user          | Логин пользователя, запросившего код                     | `manager1`           |
| phone         | Телефон, на который был запрошен код                     | `9998887766`         |
| marketplace   | Платформа входа (`Ozon`, `WB`, `Yandex`)                 | `Ozon`               |
| time_request  | Время запроса кода                                       | `2024-04-01 14:20:00`|
| time_response | Время получения кода (может быть NULL)                   | `2024-04-01 14:21:05`|
| message       | Полученный код (может быть NULL, если не пришёл)         | `123456`             |

---

## 7. 🔐 Таблицы `secret_key` и `version`

### `secret_key`

Хранит ключ шифрования, используемый для хранения логинов/паролей в `credentials.json`:

| Поле | Назначение                                | Пример                                  |
|------|-------------------------------------------|-----------------------------------------|
| key  | Ключ для шифрования логинов и паролей     | `3oxB1H0hYd7J2fwnRje6Aw==`              |

### `version`

Хранит текущую версию и URL для обновления:

| Поле   | Назначение                          | Пример                                   |
|--------|-------------------------------------|------------------------------------------|
| version| Текущая версия приложения           | `1.0.0`                                  |
| url    | Ссылка на zip-файл для обновления   | `https://21.123.23.43:2613/download_app` |
